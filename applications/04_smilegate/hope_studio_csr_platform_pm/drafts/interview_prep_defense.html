면접 방어 전략 (Interview Defense Logic)

스마일게이트 희망스튜디오 CSR 플랫폼 PM

핵심 마인드셋
"나는 배우러 온 신입이 아니라, 현장의 문제를 해결하러 온 전문가다."
윽엑윽엑하지 않는 유일한 방법은 질문을 나의 홈그라운드(도메인 지식)로 끌고 오는 것뿐입니다.
🛡️ 1. 약점 방어 (Defense Logic)

가장 아픈 질문들이 들어올 때, 변명하지 않고 '관점'을 바꿔서 받아치는 논리입니다.

Q1. "경력 6년 이상이 필수인데, 5년차시고 그마저도 기획 직무는 아니셨네요?"
[의도] Jira, Confluence 같은 툴 사용 능력이나 방법론 부재 검증
인정
"맞습니다. IT 기업의 정통 서비스 기획자 커리어는 아닙니다."
전환
"하지만 저는 기획자가 모르는 '비영리 현장'의 오퍼레이션을 압니다."
근거
"시스템은 결국 현장의 맥락을 담아야 합니다. 저는 기부금 영수증 처리가 왜 늦어지는지, 사회복지사가 어디서 좌절하는지 압니다. 툴은 배우면 되지만, 현장의 맥락은 경험 없이는 모릅니다."
Q2. "High-Q ERP는 솔루션 도입이지, 직접 바닥부터 기획한 건 아니지 않나요?"
[의도] 기획력 검증 (단순 운영만 한 것 아닌가?)
정의
"네, 솔루션 도입 맞습니다. 하지만 '우리 조직에 맞게 뜯어고치는 과정'이 진짜 기획이었습니다."
상세
"현업 용어(상환)와 시스템 용어(입금)의 갭을 메우기 위해 모든 화면 필드를 재정의하고 매핑했습니다. 이 설계 과정이 서비스 기획의 본질인 '요구사항 구체화'와 다르지 않습니다."
Q3. "말씀하시는 게 기획자보다는 개발자나 운영자 같네요."
[의도] 커뮤니케이션 스타일과 직무 적합성 확인
긍정
"그렇게 보셨다면 다행입니다. 그게 제 무기니까요."
이유
"기획자는 통역사입니다. 저는 IoT 개발 경험으로 개발자의 언어를 알고, 사회복지사 경험으로 현장의 언어를 압니다. 양쪽의 제약사항을 이해하고 조율할 수 있는 기획자, 희망스튜디오에 꼭 필요하다고 확신합니다."
⚔️ 2. 필살기 (Attack Logic)

면접관에게 끌려다니지 말고, 내가 먼저 질문을 던져서 주도권을 잡으세요.

카드 1: "임팩트 시각화" (포트폴리오 P5 연계)
"제가 포트폴리오 5페이지에 제안드린 '임팩트 대시보드' 말인데요. 제가 IoT 센서 데이터 시각화할 때 썼던 로직을 적용하면 개발 공수를 절반으로 줄일 수 있을 것 같은데, 현재 내부 데이터 구조는 어떻게 되어 있나요?"
👉 효과: 입사 후 바로 일할 사람처럼 보임 + 기술적 이해도 어필
카드 2: "게이미피케이션의 한계"
"스토브(Stove) 포인트 연동은 훌륭하지만, 기부자는 포인트 소진보다 '서사'에 반응합니다. 혹시 로스트아크 모코코 캐릭터를 활용한 인터랙티브 캠페인도 기획 범위에 포함되어 있나요?"
👉 효과: 게임 산업 이해도 + 덕력 어필 (면접관이 신나서 대답하게 만듦)
💊 3. 긴급 처방 (Panic Button)

면접장 들어가기 직전, 마인드 컨트롤용 세 문장입니다.

"잠시만요, 정리해서 말씀드리겠습니다." 말 꼬일 것 같으면 바로 끊고 3초 숨 고르기. 오히려 '신중하다'고 좋아함.
"제가 이해한 바로는 ~가 맞나요?" 동문서답 방지. 질문 다시 정의하면서 시간 벌기.
"기술적인 부분은 제가 부족할 수 있습니다.
하지만 이 문제는 이렇게 해결해보겠습니다." 모르는 건 쿨하게 인정하고, '해결하려는 태도'로 커버하기.
