## #3 Go/No-Go

### ✅ **STRONG GO**

**Go 근거:**
1. **AI 프로젝트 직결** → "전사 AI 활용 지원"이 주요 프로젝트. Claude/ChatGPT 헤비유저로서 실무 관점 기여 가능
2. **QA 실전 경험** → 맨지온에서 UI/UX 테스트 시나리오 설계·품질보증 주도 = JD 우대사항 정확히 충족
3. **ERP 도입 = 내부 서비스 기획 PM 경험** → 요구정의→설계→검수 풀사이클 = 서비스 기획 프로세스 동일
4. **웹서비스 구조 이해** → ERP 웹시스템 + AI 활용 웹 포트폴리오 기획·제작
5. **문서화 역량** → 전략서·기획서·보고서 작성 일상적
6. **포트폴리오 제출 가능** → 기존 ncsoft.html 포트폴리오 페이지 활용
7. **기존 NC 지원 데이터 재활용** → 추가 비용 0

**약점:**
- Figma 미경험 → Notion으로 유사 협업 경험 방어
- 서비스 기획 직함 없음 → "ERP 도입 PM = 내부 서비스 기획" 프레임

## #4 Positioning

### 메인 컨셉: "AI 실전 활용 + QA + 서비스 기획 PM 경험 보유 어시스턴트"

### 3축 포지셔닝
1. **AI 헤비유저**: Claude/ChatGPT를 업무자동화·문서작성·코드생성에 일상 활용. "전사 AI 활용 지원" 프로젝트에 **사용자 관점의 실전 인사이트** 제공 가능
2. **QA 실전 경험**: 맨지온에서 사용자 관점 UI/UX 테스트 시나리오 설계 주도 → 서비스 매뉴얼 작성·QA·운영에 즉시 투입
3. **내부 서비스 기획 PM**: ERP 도입 프로젝트에서 요구정의→프로세스 설계→검수 풀사이클 수행 = 서비스 기획 프로세스 직결
