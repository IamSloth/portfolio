## 2. 프로젝트 수행이력

### 프로젝트 1

| 항목 | 입력값 |
|------|--------|
| 프로젝트명 | AI 에이전트 기반 문서 자동화 파이프라인 구축 |
| 시작일 | 2026-01 |
| 종료일 | (진행 중) |
| 프로젝트 유형 | 개인 프로젝트 |
| 소속회사명 | — |
| 주사용기술 | 프롬프트 엔지니어링, Claude/ChatGPT, Python, Git |

**프로젝트 소개 (1500자 이내):**

생성형 AI(Claude, ChatGPT)를 활용한 문서 자동화 파이프라인을 개인 프로젝트로 구축·운영하고 있습니다.

[프로젝트 배경]
채용공고 분석, 이력서/자기소개서/포트폴리오 작성 등 반복적이면서도 높은 품질이 요구되는 문서 작업을 효율화하기 위해, AI 에이전트 기반의 체계적인 워크플로우를 설계했습니다.

[수행 역할]
1. 시스템 프롬프트 설계: AI 에이전트가 일관된 품질의 결과물을 생성하도록 가이드라인·규칙·워크플로우를 정의한 200줄 이상의 시스템 프롬프트를 작성하고 지속적으로 최적화했습니다. 파일명 규칙, 상태 코드, 템플릿 레지스트리, 난이도별 AI 모델 라우팅 등 데이터 관리 체계도 설계했습니다.

2. AI 출력물 품질 평가 및 피드백: AI가 생성한 문서를 "정확성-적합성-자연스러움" 3가지 기준으로 평가하고, 구체적인 개선 피드백을 반복 수행했습니다. 수백 회의 평가-피드백 사이클을 통해 AI 출력물의 품질을 좌우하는 핵심 요인(프롬프트 설계, 가이드라인 명확성, 맥락 제공)을 체득했습니다.

3. 프롬프트 체인 설계: 채용공고 입력 → JD 분석 → 적합도 매칭 → 전략 수립 → 문서 생성까지 일관된 프로세스가 자동 실행되도록 멀티스텝 프롬프트를 구조화했습니다.

4. LLM 활용 코드 실행: Python 스크립트를 AI와 협업하여 작성·실행하고, .hwpx(한컴) 문서 자동 채우기, base64 이미지 추출, 데이터 가공 등의 작업을 수행했습니다.

[성과]
- 27개 기업 지원 문서를 일관된 품질로 생성
- 프롬프트 최적화를 통해 문서 작성 시간 대폭 단축 및 재작업률 감소

이 프로젝트를 통해 AI 출력물의 품질을 좌우하는 핵심 원리—가이드라인 설계, 데이터 품질 관리, 체계적 평가—를 실전에서 체득했으며, 이는 AI 데이터 라벨링 업무의 본질과 직결됩니다.

- 인프라 예산 약 3.6억 원의 체계적 관리 기반 구축

이 프로젝트에서 "대량의 데이터를 분류하고 기준을 수립하여 품질을 관리하는" 역량과, "현업의 요구사항을 명확한 가이드라인으로 변환하는" 역량을 쌓았습니다.

## 3. 자기소개 (2500자 이내)

지난 3개월간 매일 Claude와 ChatGPT를 활용해 업무를 수행했습니다. 채용공고 분석, 이력서 작성, 데이터 추출, 문서 자동화까지—AI가 할 수 있는 것과 할 수 없는 것의 경계를 직접 체감하며 일했습니다.

같은 질문이라도 프롬프트를 어떻게 설계하느냐에 따라 출력 품질이 극적으로 달라졌고, AI가 생성한 결과물을 다양한 관점에서 평가하고 수정하는 과정을 수백 회 반복했습니다. 이 과정에서 "AI가 정확한 답을 내기 위해서는 양질의 데이터와 명확한 기준이 필수"라는 것을 깊이 체감했습니다. 카카오의 AI 검색 조직이 "수많은 사용자의 검색 여정 속에서 가장 정확하고 만족스러운 답을 제공"하기 위해 데이터의 품질을 관리하는 일—그것이 제가 하고 싶은 일입니다.

[프롬프트 엔지니어링과 가이드라인 설계]

AI를 단순히 사용하는 데 그치지 않았습니다. 200줄 이상의 시스템 프롬프트를 직접 설계하여, AI 에이전트가 채용공고 분석→적합도 평가→전략 수립→문서 생성까지 일관된 프로세스를 자동 수행하도록 구조화했습니다. 파일명 규칙, 상태 코드, 품질 기준, 난이도별 AI 모델 선택(라우팅) 등 세부 가이드라인을 정의하고 수백 회의 실험을 거쳐 최적화했습니다.

이 과정에서 체득한 핵심 인사이트가 있습니다. 모호한 지시는 모호한 결과를, 구체적이고 구조화된 지시는 일관된 고품질 결과를 만든다는 것입니다. 예를 들어, "이력서를 써줘"라는 프롬프트와 "채용공고의 필수 키워드를 반영하고, 성과를 정량적으로 기술하며, 500자 이내로 작성해줘"라는 프롬프트의 결과물 차이는 극적입니다. 가이드라인의 명확성이 곧 데이터의 품질을 결정한다는 이 원칙은, 어노테이션 가이드라인 수립 업무에 그대로 적용할 수 있습니다.

[QA 관점의 품질 평가 역량]

(주)맨지온에서 임베디드 시스템의 QA를 담당하며, 사용자 관점의 테스트 시나리오를 설계하고 품질 보증 활동을 주도한 경험이 있습니다. "기술적으로 동작하는가"뿐 아니라 "사용자가 기대하는 결과를 제공하는가"를 기준으로 품질을 판단하는 시각을 갖추고 있습니다.

이 QA 마인드를 AI 출력물 평가에도 적용하고 있습니다. AI가 생성한 결과가 사실에 기반하는지(정확성), 질문의 의도에 부합하는지(적합성), 표현이 자연스러운지(유창성)를 다각도로 평가할 수 있습니다. IoT 모니터링 시스템에서 센서 데이터의 이상치를 탐지하고 통신 장애를 분석한 경험도, 대량 데이터의 품질을 꼼꼼하게 검수하는 역량의 토대가 되었습니다.

[데이터 관리와 프로세스 설계]

(사)함께만드는세상에서 3년간 총무·IT 지원을 담당하며, 커스텀 ERP 도입 프로젝트에서 요구사항 정의→프로세스 설계→계정과목 표준화→데이터 검증→검수까지 풀사이클을 수행했습니다. 부서별 업무 프로세스를 분석하여 시스템 요구사항으로 변환하고, 마이그레이션 데이터의 정합성을 검증하는 과정에서 "대량의 데이터를 분류하고, 기준을 수립하여 품질을 관리하는" 역량을 쌓았습니다. 이는 AI 학습 데이터의 체계적 관리와 본질이 같습니다.

[다양한 도메인 교차 경험과 코드 이해력]

전기공학(HW개발)→임베디드(IoT/QA)→사회복지(대학원)→IT인프라(ERP/총무)→AI 활용까지, 다양한 분야를 거치며 새로운 도메인을 빠르게 학습하고 복잡한 정보를 체계적으로 정리하는 역량을 키워왔습니다. AI 검색 서비스가 다루는 데이터 역시 다양한 도메인에 걸쳐 있기에, 이 폭넓은 이해력이 데이터 품질 판단에 강점이 됩니다.

또한 전기공학 전공과 임베디드 개발(ESP32/C++) 경험을 바탕으로 코드를 읽고 이해하는 데 어려움이 없으며, LLM과 협업하여 Python 스크립트를 작성·실행하며 문서 자동화, 데이터 가공 등의 작업을 일상적으로 수행하고 있습니다.

카카오의 AI 검색 서비스에서 데이터의 품질이 곧 서비스의 품질이라는 믿음으로, 꼼꼼하고 책임감 있게 기여하겠습니다.
