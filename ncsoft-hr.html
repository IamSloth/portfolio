PROJECT 01
ERP 도입 — 데이터 추출·가공의 원점
2023.07 → 2024.12
200여 개 계정과목을 엑셀로 수기 관리하던 환경에서, Tibero DB 기반 커스텀 ERP를 요구사항 정의부터 데이터 검수·마이그레이션까지 단독으로 수행한 프로젝트입니다. SQL 직접 작성, 데이터 정합성 검증, 피벗·Power Query 기반 보고서 체계를 구축했습니다.
Phase 1 — 데이터 현황 분석
기존 엑셀 원장 전수 분석. 200개 계정과목 재분류·표준화. 부서별·사업별 매핑 테이블 설계.
Phase 2 — SQL 데이터 추출·검수
Tibero DB 직접 쿼리 작성. 마이그레이션 전후 데이터 정합성 검증. 불일치 항목 식별 및 수정.
Phase 3 — Excel 보고서 자동화
Pivot Table·Power Query로 ERP 추출 데이터를 부서별 집행 현황 보고서로 자동 변환. 월말 결산 리드타임 단축.
Phase 4 — 운영·안정화
관리자 권한으로 시스템 운영. 사용자 문의 대응 및 데이터 오류 수정. 연간 예산 집행률 88.5% 달성.
BEFORE
엑셀 수기 관리 → 분기 결산 5일 소요
부서별 집행 현황 파악 불가
데이터 추출 = 수작업 복사·붙여넣기
AFTER
SQL 직접 추출 → 결산 리드타임 2일 단축
부서별·사업별 집행률 실시간 조회
보고서 자동화로 반복 가공 작업 제거
예산 집행률 88.5% · 결산 리드타임 2일 단축
PROJECT 02
전사 커뮤니케이션 허브 — 일 수십 건 문의 처리
2023.01 → 2026.02
IT인프라·총무 담당자로서 전 부서의 문의·요청이 단일 창구로 집중되는 구조에서, 8개 외주업체 조율 + 전부서 요청 처리 + 프로세스 표준화를 3년간 수행한 사례입니다. 반복 문의를 FAQ 매뉴얼로 정리해 대응 시간을 구조적으로 단축했습니다.
수신 — 전부서 문의 단일 창구
PC·네트워크·복합기·계약·비품 등 전부서 일상 요청 접수. 우선순위 분류 후 즉시 처리 또는 외주 연결.
조율 — 외주 8개사 실시간 관리
유지보수·렌탈·보안·청소 등 업체별 SLA 관리. 장애 발생 시 에스컬레이션 판단 및 현장 대응 지원.
표준화 — FAQ 매뉴얼 구축
반복 문의 유형 분류 및 대응 절차 문서화. 신규 담당자 온보딩 가이드 작성. 구두 전달 → 문서 체계 전환.
BEFORE
문의 유형 미분류 → 매번 처음부터 대응
외주 연락처 분산 → 처리 지연
담당자 교체 시 노하우 소실
AFTER
FAQ 매뉴얼 → 반복 문의 즉시 처리
외주 8개사 단일 연락 체계 구축
문서화로 인수인계 리스크 제거
