프로젝트 상세

PROJECT 01
ERP 도입 = 내부 서비스 기획 PM
2023.07 → 2024.12
200여 개 예산 항목을 엑셀로 수기 관리하던 환경에서, Tibero DB/Nexacro 기반 ERP를 사용자 요구정의 → 프로세스 설계 → 데이터 마이그레이션 → 인수 테스트까지 단독으로 수행한 내부 서비스 기획 프로젝트입니다. 외부 SaaS가 아닌 커스텀 시스템을 현업 담당자로서 기획·검수한 경험은, 서비스 기획 PM의 전형적인 풀사이클과 동일한 구조입니다.
Phase 1 — 요구사항 수집 (User Research)
현업 부서 인터뷰 → 예산·회계 프로세스 흐름 파악. 기존 엑셀 관리의 Pain Point(중복 입력, 추적 불가, 결산 지연) 문서화 및 우선순위 정의.
Phase 2 — 프로세스 설계 (Service Design)
200개 예산 항목 재분류. 부서별·사업별 매핑 테이블 설계. 사용자 플로우 정의 및 개발사와 요구사항 협의.
Phase 3 — 데이터 마이그레이션 (Data Migration)
SQL 쿼리로 마이그레이션 데이터 정합성 검증. 불일치 항목 식별·수정. 기존 데이터의 신규 스키마 매핑 완료.
Phase 4 — 인수 테스트 (Acceptance Testing)
전사 기능 테스트 시나리오 직접 작성·실행. 버그 리포트 관리. 현업 담당자 관점의 UAT 수행 후 안정화.
BEFORE
엑셀 수기 관리 → 분기 결산 5일 소요
부서별 집행 현황 파악 불가
오류 발생 시 추적 불가능
AFTER
ERP 실시간 조회 → 결산 2일 단축
부서별·사업별 집행률 즉시 확인
데이터 기반 의사결정 환경 구축
요구정의→설계→검수 풀사이클 단독 수행
PROJECT 02
IoT 시스템 QA — 사용자 관점 테스트 설계
2021.04 → 2022.06
맨지온에서 ESP32 기반 IoT 모니터링 시스템의 UI/UX 테스트 시나리오 설계부터 버그 리포트 프로세스 수립까지 담당했습니다. 개발자 시각이 아닌 실제 사용자 관점에서 테스트 케이스를 설계하는 QA 방법론을 실전에서 체득한 프로젝트입니다.
UI/UX 테스트 시나리오 설계
사용자 시나리오 기반 테스트 케이스 작성. 엣지 케이스·예외 상황 목록화. 기능 명세서와 실제 동작 간 GAP 분석.
버그 리포트 프로세스 수립
버그 재현 절차·우선순위·심각도 기준 문서화. 개발팀과 버그 트래킹 워크플로우 정립. 이슈 분류 체계 설계.
데이터 로그 분석 기반 트러블슈팅
센서 데이터 로그 분석으로 간헐적 오류 패턴 파악. 재현 조건 정의 후 개발팀에 전달. 검증 완료까지 추적.
BEFORE
애드혹(Ad-hoc) 테스트
버그 재현 조건 불명확
개발·QA 소통 비효율
AFTER
체계적 QA 프로세스 정립
버그 리포트 표준화 → 해결 속도 향상
사용자 관점 테스트 문화 정착
Ad-hoc → 구조화된 QA 프로세스 전환
PROJECT 03
AI 업무 활용 — 전사 AI 프로젝트 사용자 관점 기여
2024 → Current
Claude, ChatGPT를 매일 실무에 적용하는 헤비유저입니다. 문서 초안 작성, 코드 생성, 데이터 분석, 워크플로우 자동화까지 — AI 도구의 실전 활용 패턴을 직접 구축했습니다. NCsoft의 전사 AI 활용 지원 프로젝트에서 "실제로 써본 사람"의 관점을 가장 직접적으로 기여할 수 있습니다.
문서 초안 생성 자동화
이력서·자소서·기획서 초안을 Claude로 생성 → 반복 피드백 루프로 품질 개선. 프롬프트 설계·최적화 경험 축적.
코드·스크립트 생성 및 검증
Python 자동화 스크립트, HTML/CSS/JS 코드 생성에 AI 적극 활용. 생성된 코드 검증 및 디버깅 실전 경험 보유.
데이터 분석 및 워크플로우 설계
Excel 데이터 분석, SQL 쿼리 최적화, 업무 프로세스 재설계에 AI 보조 활용. 비개발직군의 AI 활용 한계와 가능성을 실전에서 파악.
AI 활용 포트폴리오 직접 구현
이 페이지를 포함한 웹 포트폴리오를 AI 협업으로 직접 설계·개발. HTML/CSS/JS 기반 Awwwards 수준 디자인 구현.
